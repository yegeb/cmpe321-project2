\section*{Tasks \& Contributions}

I served as the primary backend and frontend architect. I established the entire Express.js infrastructure (routing, middleware, authentication, database layer), designed and implemented user interface panels for all four roles (Database Manager, Manager, Player, Referee), and authored all SQL queries for Manager and Player roles.

\textbf{Manager role:} League standings queries (3 points/win, 1 point/draw), squad statistics, competition leaderboards (top scorers, assists, ratings). \textbf{Player role:} Career aggregation, seasonal/competition filtering, match history, transfer/contract records.

I conducted comprehensive edge-case testing of my partner Tuğçe's Database Manager and Referee features (stadium conflicts, contract limits, loan parent validation), discovering and fixing several UI and backend inconsistencies. I identified and resolved a critical constraint that was only enforced in the backend: goal consistency for match results (player goal totals must sum to team score). Following the specification's strict enforcement rule, I moved this validation to a MySQL trigger (\texttt{trg\_match\_result\_goal\_consistency}) to ensure the database itself protects integrity regardless of how data is inserted.
